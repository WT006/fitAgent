## 1. 项目概述

### 1.1 产品定位

fitagent 是一款面向大众的运动健康 AI 助手。用户通过与 AI 的引导式问答，生成个性化的 **30 天健康计划**，并在打卡页中逐日完成任务、标记完成情况。

### 1.2 目标用户

以**运动新手**和**日常维持者**为主——计划简单、重在坚持的人群。产品应让没有运动基础的用户也能快速上手。

### 1.3 技术架构

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 |
| 后端 | Spring Boot（在现有项目上扩展） |
| 数据库 | MySQL（用户、计划、任务）；对话记忆暂用 Redis，后续迁移 MySQL |
| 架构模式 | C/S 架构 |

### 1.4 已有功能（本期不改动）

| 模块 | 说明 |
|------|------|
| AI 对话 | 多轮会话 + RAG 运动健康知识库（`FitApp`） |
| 超级智能体 | 支持工具调用的自主规划执行（`Manus`） |

## 2. 本期新增功能

本期 MVP 新增三个模块：**用户登录**、**30 天打卡**、**AI 规划**。

---

### 2.1 用户登录

| 功能 | 说明 |
|------|------|
| 注册 | 用户名 + 密码，密码加密存储 |
| 登录 | 登录后获取 Token，后续请求需携带 |
| 退出 | 清除登录状态 |
| 数据隔离 | 每个用户拥有独立的 30 天计划，互不可见 |

---

### 2.2 30 天打卡页

打卡页是本期核心页面，**替代原日历方案**。

#### 2.2.1 进度展示

- 显示当前第几天 / 共 30 天（如：第 8 天 / 30 天）
- 进度条可视化

#### 2.2.2 今日任务

- 展示今天 3～5 条任务
- 任务类型分为四类：**运动**、**饮食**、**休息**、**习惯**
- 点击任务可标记「已完成」，再次点击恢复「未完成」

#### 2.2.3 查看其他日期

| 日期范围 | 行为 |
|----------|------|
| 过去 | 可查看任务列表，支持补打勾 |
| 今天 | 可查看并打勾 |
| 未来 | 可预览任务，不可提前打勾 |

#### 2.2.4 计划约束

- 每个用户同时只有**一套**进行中的 30 天计划
- 不支持同时维护多套计划

---

### 2.3 AI 规划

#### 2.3.1 入口

打卡页提供 **「AI 帮我规划」** 按钮，点击后跳转至 AI 对话页，进入**规划模式**（非普通闲聊）。

#### 2.3.2 交互流程

```
打卡页点击「AI 帮我规划」
    ↓
跳转 AI 对话页（规划模式）
    ↓
AI 按固定问题依次提问，用户逐条回答
    ↓
AI 根据回答生成从今天起的 30 天每日任务
    ↓
弹出预览，用户确认
    ↓
自动跳转回打卡页
```

#### 2.3.3 固定问题（MVP）

AI 按以下顺序提问，用户答完一题再问下一题：

| 序号 | 问题 |
|------|------|
| 1 | 你每天大概有多少时间？ |
| 2 | 你的目标是什么？（减脂 / 增肌 / 保持习惯） |
| 3 | 有没有身体限制？（选填） |
| 4 | 需要安排饮食或作息吗？ |

#### 2.3.4 计划生成规则

- 从今天开始，连续生成 30 天的每日任务
- 每天 3～5 条简单打卡项，类型为运动 / 饮食 / 休息 / 习惯
- 生成后展示预览，用户可勾选、修改标题，确认后批量写入

#### 2.3.5 重新规划

- 用户**随时**可再次点击「AI 帮我规划」
- 确认重新规划后：
  - 旧计划作废
  - 从今天起重新生成新的 30 天计划
  - 历史打卡记录一并清除

#### 2.3.6 与普通对话的区别

| 对比项 | 普通 AI 对话 | AI 规划模式 |
|--------|-------------|-------------|
| 入口 | 导航栏「AI 对话」 | 打卡页「AI 帮我规划」 |
| 交互方式 | 自由聊天 | 固定问题引导 |
| 目的 | 运动健康咨询答疑 | 生成 30 天打卡计划 |
| 结果 | 文字回复 | 写入打卡计划并跳转回打卡页 |

## 3. MVP 范围

### 3.1 包含功能

- [x] 用户注册 / 登录 / 退出
- [x] 30 天打卡页（进度、今日任务、历史补打勾、未来预览）
- [x] AI 规划模式（固定问答 → 生成 30 天计划 → 预览确认 → 自动回打卡页）
- [x] 随时重新规划（旧计划作废）
- [x] 已有 AI 对话 + RAG 知识库
- [x] 已有 Manus 超级智能体

### 3.2 不包含功能

| 排除项 | 说明 |
|--------|------|
| 日历月视图 | 本期用 30 天打卡替代 |
| 推送提醒 | 后续版本考虑 |
| 可穿戴设备接入 | 后续版本考虑 |
| 社交 / 排行榜 | 不做 |
| 营养计算 / 卡路里追踪 | 不做 |
| 统计图表 / 连续打卡天数 | 本期仅进度条 + 第几天 |
| 结构化训练记录 | 不做组数、重量等细节 |
| 同时多套计划 | 一人始终只有一套 |
| 自由聊天后写入计划 | 规划走固定问答，不走自由聊天 |

## 4. 数据设计（概要）

### 4.1 用户表 `user`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| username | VARCHAR(50) | 登录名，唯一 |
| password | VARCHAR(255) | 加密密码 |
| nickname | VARCHAR(50) | 显示昵称 |
| created_at | DATETIME | 注册时间 |

### 4.2 计划表 `plan`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 所属用户 |
| start_date | DATE | 计划开始日期（今天） |
| status | TINYINT | 0=进行中, 1=已完成, 2=已作废 |
| created_at | DATETIME | 创建时间 |

### 4.3 每日任务表 `daily_task`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| plan_id | BIGINT | 所属计划 |
| day_index | INT | 第几天（1～30） |
| task_date | DATE | 任务日期 |
| title | VARCHAR(200) | 任务标题 |
| type | VARCHAR(20) | SPORT / DIET / REST / HABIT |
| status | TINYINT | 0=未完成, 1=已完成 |
| completed_at | DATETIME | 完成时间（可空） |

## 6. 验收标准

| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| AC-01 | 新用户注册并登录 | 进入打卡页 |
| AC-02 | 点击「AI 帮我规划」并完成问答 | AI 生成 30 天计划，确认后自动回到打卡页 |
| AC-03 | 查看今日任务并打勾 | 状态变为已完成 |
| AC-04 | 查看昨天任务并补打勾 | 可正常标记 |
| AC-05 | 查看未来某天 | 可预览，不可打勾 |
| AC-06 | 重新点击「AI 帮我规划」并确认 | 旧计划作废，从今天生成新 30 天，历史记录清除 |
| AC-07 | 用户 A 与用户 B 数据 | 互不可见 |
| AC-08 | 未登录访问接口 | 返回 401 |

## 8. 附录：需求演进记录

| 阶段 | 决策 |
|------|------|
| 初始想法 | 日历规划 + 事后标记完成 |
| 调整 1 | AI 写入改为独立入口 + 固定问答引导 |
| 调整 2 | 规划结束后自动跳回打卡页 |
| 最终决策 | 放弃日历，改为 30 天打卡；支持随时重新规划（旧计划作废） |

---

**一句话总结**：点「AI 帮我规划」→ 回答固定问题 → 生成 30 天任务 → 每天打勾；随时可重新规划，旧计划作废。
