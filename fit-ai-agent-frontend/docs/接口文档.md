## 1. 通用说明

### 1.1 统一响应格式

所有 JSON 接口（`/user/**`、`/plan/**`）均返回以下结构：

```json
{
  "code": 0,
  "data": {},
  "message": "ok"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | `0` 表示成功，非 0 表示失败 |
| data | object / array / null | 业务数据 |
| message | string | 提示信息 |

### 1.2 常见错误码

| code | 说明 |
|------|------|
| 0 | 成功 |
| 40000 | 请求参数错误 |
| 40100 | 未登录 |
| 40101 | 无权限 |
| 40400 | 数据不存在 |
| 50001 | 操作失败 |

### 1.3 鉴权方式

需要登录的接口，请求头携带：

```
Authorization: Bearer <token>
```

登录成功后，`/user/login` 返回的 `data.token` 即为 JWT Token，有效期 7 天。

### 1.4 接口鉴权范围

| 路径 | 是否需要登录 |
|------|-------------|
| `/user/login`、`/user/register` | 否 |
| `/user/**`（其余） | 是 |
| `/plan/**` | 是 |
| `/fitai/**` | 否（已有功能，本期不改动） |

### 1.5 日期格式

日期参数统一使用 ISO 格式：`yyyy-MM-dd`，例如 `2026-07-13`。

## 2. 用户模块

### 2.1 用户注册

- **URL**：`POST /user/register`
- **鉴权**：不需要
- **说明**：注册新用户

**请求体**

```json
{
  "userAccount": "testuser",
  "userPassword": "12345678",
  "checkPassword": "12345678",
  "userName": "测试用户"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userAccount | string | 是 | 账号，4-16 位，字母/数字/下划线 |
| userPassword | string | 是 | 密码，至少 8 位 |
| checkPassword | string | 是 | 确认密码，需与密码一致 |
| userName | string | 否 | 昵称，不填则默认使用账号 |

**响应 data**

```json
1234567890123456789
```

返回新注册用户的 `id`（number）。

---

### 2.2 用户登录

- **URL**：`POST /user/login`
- **鉴权**：不需要
- **说明**：登录并获取 Token

**请求体**

```json
{
  "userAccount": "testuser",
  "userPassword": "12345678"
}
```

**响应 data**

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "userVO": {
    "id": 1234567890123456789,
    "userAccount": "testuser",
    "userName": "测试用户",
    "userRole": "user",
    "createTime": "2026-07-13T10:00:00"
  }
}
```

**前端处理建议**：将 `token` 存入 localStorage，后续请求统一带上 `Authorization` 头。

---

### 2.3 用户退出

- **URL**：`POST /user/logout`
- **鉴权**：需要
- **说明**：服务端无状态，客户端删除 Token 即可

**请求体**：无

**响应 data**

```json
true
```

**前端处理建议**：清除本地 Token，跳转登录页。

---

### 2.4 获取当前登录用户

- **URL**：`GET /user/get/login`
- **鉴权**：需要
- **说明**：获取当前登录用户信息，用于页面初始化

**响应 data**

```json
{
  "id": 1234567890123456789,
  "userAccount": "testuser",
  "userName": "测试用户",
  "userRole": "user",
  "createTime": "2026-07-13T10:00:00"
}
```

## 3. 打卡模块

### 3.1 获取当前计划进度

- **URL**：`GET /plan/current`
- **鉴权**：需要
- **说明**：打卡页顶部进度条数据

**响应 data（有进行中的计划）**

```json
{
  "planId": 1234567890123456789,
  "startDate": "2026-07-13",
  "currentDay": 8,
  "totalDays": 30,
  "progressPercent": 26,
  "hasPlan": true
}
```

**响应 data（无计划）**

```json
{
  "planId": null,
  "startDate": null,
  "currentDay": 0,
  "totalDays": 30,
  "progressPercent": 0,
  "hasPlan": false
}
```

| 字段 | 说明 |
|------|------|
| currentDay | 当前第几天（1-30） |
| progressPercent | 进度百分比，用于进度条 |
| hasPlan | 是否有进行中的计划 |

---

### 3.2 获取指定日期任务列表

- **URL**：`GET /plan/tasks?date=2026-07-13`
- **鉴权**：需要
- **说明**：查看某天任务，支持今天 / 过去 / 未来

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 是 | 日期，格式 `yyyy-MM-dd` |

**响应 data**

```json
{
  "date": "2026-07-13",
  "dayIndex": 8,
  "canToggle": true,
  "tasks": [
    {
      "id": 111,
      "dayIndex": 8,
      "title": "快走 20 分钟",
      "type": "SPORT",
      "status": 0,
      "completedAt": null
    },
    {
      "id": 112,
      "dayIndex": 8,
      "title": "喝够 8 杯水",
      "type": "HABIT",
      "status": 1,
      "completedAt": "2026-07-13T18:30:00"
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| dayIndex | 第几天（1-30） |
| canToggle | 是否允许打勾。今天及过去为 `true`，未来为 `false` |
| tasks[].type | 任务类型，见下方枚举 |
| tasks[].status | `0`=未完成，`1`=已完成 |

**任务类型枚举**

| type | 含义 |
|------|------|
| SPORT | 运动 |
| DIET | 饮食 |
| REST | 休息 |
| HABIT | 习惯 |

---

### 3.3 切换任务完成状态

- **URL**：`POST /plan/task/toggle`
- **鉴权**：需要
- **说明**：点击任务打勾 / 取消打勾

**请求体**

```json
{
  "taskId": 111
}
```

**响应 data**：返回更新后的任务对象

```json
{
  "id": 111,
  "dayIndex": 8,
  "title": "快走 20 分钟",
  "type": "SPORT",
  "status": 1,
  "completedAt": "2026-07-13T18:30:00"
}
```

**业务规则**

- 仅今天及过去的任务可打勾
- 未来日期调用会返回错误：`未来日期不可提前打勾`

## 4. AI 规划模块

规划模式通过固定 4 个问题引导用户，AI 生成 30 天计划预览，用户确认后写入打卡计划。

### 4.1 固定问题（供前端展示参考）

| 序号 | 问题 |
|------|------|
| 1 | 你每天大概有多少时间？ |
| 2 | 你的目标是什么？（减脂 / 增肌 / 保持习惯） |
| 3 | 有没有身体限制？（选填，没有可回复「无」） |
| 4 | 需要安排饮食或作息吗？ |

---

### 4.2 规划模式对话

- **URL**：`POST /plan/ai/chat`
- **鉴权**：需要
- **说明**：进入规划模式或提交用户回答

**请求体**

```json
{
  "message": "每天大概 30 分钟"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 视情况 | 首次进入可留空；回答问题时必填 |

**响应 data（提问阶段）**

```json
{
  "phase": "question",
  "questionIndex": 1,
  "question": "你每天大概有多少时间？",
  "preview": null
}
```

**响应 data（预览阶段）**

```json
{
  "phase": "preview",
  "questionIndex": null,
  "question": null,
  "preview": {
    "days": [
      {
        "dayIndex": 1,
        "tasks": [
          { "title": "快走 15 分钟", "type": "SPORT" },
          { "title": "午餐少油少盐", "type": "DIET" },
          { "title": "23:00 前入睡", "type": "REST" }
        ]
      }
    ]
  }
}
```

| phase | 说明 |
|-------|------|
| question | 正在提问，展示 `question` 给用户 |
| preview | 问答完成，展示 `preview` 供用户确认/编辑 |

**前端调用流程**

```
1. POST /plan/ai/chat {}              → 获取第 1 题
2. POST /plan/ai/chat {message}       → 提交回答，获取下一题
3. 重复步骤 2，共 4 次
4. 第 4 次回答后 phase=preview        → 展示预览页
5. POST /plan/ai/confirm              → 确认写入
```

---

### 4.3 获取计划预览

- **URL**：`GET /plan/ai/preview`
- **鉴权**：需要
- **说明**：重新获取当前预览（如页面刷新后恢复预览状态）

**响应 data**

```json
{
  "days": [
    {
      "dayIndex": 1,
      "tasks": [
        { "title": "快走 15 分钟", "type": "SPORT" }
      ]
    }
  ]
}
```

> 需先完成 4 题问答，否则返回 `40400 暂无计划预览，请先完成问答`。

---

### 4.4 确认写入计划

- **URL**：`POST /plan/ai/confirm`
- **鉴权**：需要
- **说明**：用户确认预览后，批量写入 30 天计划。若已有进行中的计划，会自动作废并清除历史打卡。

**请求体**

```json
{
  "days": [
    {
      "dayIndex": 1,
      "tasks": [
        { "title": "快走 15 分钟", "type": "SPORT" },
        { "title": "喝够 8 杯水", "type": "HABIT" }
      ]
    },
    {
      "dayIndex": 2,
      "tasks": [
        { "title": "深蹲 3 组", "type": "SPORT" }
      ]
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| days | 必须包含 30 天（dayIndex 1-30） |
| days[].tasks | 每天 1-5 条任务，用户可在预览页修改 title |
| tasks[].type | 必须是 `SPORT` / `DIET` / `REST` / `HABIT` |

**响应 data**：同「获取当前计划进度」接口

```json
{
  "planId": 1234567890123456789,
  "startDate": "2026-07-13",
  "currentDay": 1,
  "totalDays": 30,
  "progressPercent": 3,
  "hasPlan": true
}
```

**前端处理建议**：确认成功后跳转打卡页 `/checkin`。

---

### 4.5 重新规划

- **URL**：`POST /plan/ai/replan`
- **鉴权**：需要
- **说明**：用户再次点击「AI 帮我规划」时调用。作废旧计划、清除历史打卡、重置问答，返回第 1 题。

**请求体**：无

**响应 data**

```json
{
  "phase": "question",
  "questionIndex": 1,
  "question": "你每天大概有多少时间？",
  "preview": null
}
```

**前端处理建议**：弹出确认框「重新规划将清除当前计划及打卡记录」，用户确认后再调用此接口。

## 5. AI 对话模块（已有功能）

> 以下接口无需登录，用于普通运动健康咨询，与规划模式独立。

### 5.1 同步对话

- **URL**：`GET /fitai/fitapp/chat/sync`
- **鉴权**：不需要

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户消息 |
| chatId | string | 是 | 会话 ID，前端生成 UUID 保持多轮上下文 |

**响应**：纯文本字符串（非 JSON 包装）

```
根据你的情况，建议从低强度有氧运动开始……
```

---

### 5.2 流式对话

- **URL**：`GET /fitai/fitapp/chat/stream`
- **鉴权**：不需要
- **Content-Type**：`text/event-stream`

**Query 参数**：同同步对话

**响应**：SSE 流式文本片段，前端逐字拼接展示。

**前端示例（EventSource）**

```javascript
const url = `/api/fitai/fitapp/chat/stream?message=${encodeURIComponent(msg)}&chatId=${chatId}`
const es = new EventSource(url)
es.onmessage = (e) => { /* 拼接 e.data */ }
```

---

### 5.3 Manus 超级智能体

- **URL**：`GET /fitai/manus/chat`
- **鉴权**：不需要
- **Content-Type**：`text/event-stream`

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户消息 |

**响应**：SSE 流式事件。

## 7. 典型业务流程

### 7.1 新用户首次使用

```
注册 → 登录 → 进入打卡页（hasPlan=false）
→ 点击「AI 帮我规划」→ replan/chat 开始问答
→ 完成 4 题 → 预览 → 确认 confirm
→ 回到打卡页（hasPlan=true）→ 查看今日任务 → 打勾 toggle
```

### 7.2 老用户每日打卡

```
登录 → GET /plan/current（进度）
→ GET /plan/tasks?date=今天（今日任务）
→ POST /plan/task/toggle（打勾）
```

### 7.3 查看其他日期

```
GET /plan/tasks?date=2026-07-10
→ canToggle=true（过去，可补打勾）
→ canToggle=false（未来，只读预览）
```

### 7.4 重新规划

```
打卡页点击「AI 帮我规划」→ 确认弹窗
→ POST /plan/ai/replan
→ 重新走 4 题问答 → 预览 → confirm
→ 跳转打卡页
```

## 8. Axios 封装参考

```javascript
import axios from 'axios'

const request = axios.create({
  baseURL: 'http://localhost:8122/api',
  timeout: 30000,
})

request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

request.interceptors.response.use(
  (res) => {
    const { code, data, message } = res.data
    if (code === 0) return data
    if (code === 40100) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(new Error(message || '请求失败'))
  },
  (err) => Promise.reject(err)
)

export default request
```

## 9. 联调注意事项

1. **跨域**：后端已配置 CORS，前端开发服务器直接请求即可。
2. **规划接口耗时**：`POST /plan/ai/chat` 在第 4 题回答后会调用 AI 生成 30 天计划，可能需要 10-30 秒，前端需展示 loading。
3. **确认计划数据量**：`confirm` 请求体包含 30 天任务，体积较大，不要用 GET 缓存。
4. **会话有效期**：规划问答会话存 Redis，2 小时过期；超时需重新调用 `/plan/ai/chat` 开始。
5. **Swagger 文档**：启动后端后访问 `http://localhost:8122/api/doc.html` 可查看 Knife4j 在线文档。
