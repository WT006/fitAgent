## 1. 文档说明

### 1.1 目标

本文档为 fitagent MVP 前端的技术实现方案，覆盖从零搭建 Vue 3 SPA 到与现有 Spring Boot 后端联调的完整设计。**本文档不包含代码实现**，仅作为开发与验收的技术依据。

### 1.2 已确认的技术决策

| 编号 | 决策项 | 确认结果 |
|------|--------|----------|
| D-01 | 登录方式 | 用户名 + 密码（不含 GitHub OAuth） |
| D-02 | UI 方案 | Tailwind CSS + 自定义组件 |
| D-03 | 语言 | TypeScript |
| D-04 | AI 对话页 | 单页双模式：FitApp 咨询 + Manus 智能体 |
| D-05 | AI 规划模式 | **独立子模式**（见 §4.4 设计决策） |
| D-06 | 部署范围 | 仅本地开发联调 |

### 1.3 现状调研结论

| 层级 | 状态 |
|------|------|
| 前端 | **空白**，需从零搭建 |
| 后端 | **已实现**，Spring Boot，`http://localhost:8122/api` |
| 数据库 | MySQL（用户、计划、任务） |
| 缓存 | Redis（对话记忆、规划会话） |

后端已实现全部 MVP 接口，前端以 `docs/接口文档.md` 为契约，**本期不改动后端**。

## 2. 总体架构

### 2.1 系统架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    Vue 3 SPA (Vite Dev :5173)               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────────┐ │
│  │  Views   │→ │  Router  │→ │  Pinia   │→ │ Axios/SSE   │ │
│  └──────────┘  └──────────┘  └──────────┘  └──────┬──────┘ │
└────────────────────────────────────────────────────│────────┘
                                                     │ /api 代理
┌────────────────────────────────────────────────────▼────────┐
│              Spring Boot Backend (:8122/api)                  │
│  ┌────────────┐  ┌────────────┐  ┌────────────────────────┐ │
│  │ /user/**   │  │ /plan/**   │  │ /fitai/**              │ │
│  │ JWT 鉴权   │  │ JWT 鉴权   │  │ 无需鉴权（已有功能）    │ │
│  └─────┬──────┘  └─────┬──────┘  └───────────┬────────────┘ │
│        │               │                      │              │
│        ▼               ▼                      ▼              │
│     MySQL           MySQL + Redis          Redis + AI       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 架构原则

1. **MVP 最小化**：仅实现需求文档 §3.1 所列功能，不扩展统计、推送、日历等。
2. **后端契约优先**：前端适配已有 API，不新增后端接口（Manus PDF 下载见 §8.3 已知限制）。
3. **模式隔离**：FitApp 咨询、Manus 智能体、AI 规划三种交互模式各自独立，避免状态污染。
4. **鉴权分层**：`/user/**`、`/plan/**` 需 JWT；`/fitai/**` 无需登录（与后端一致）。

## 5. 路由设计

### 5.1 路由表

| 路由 | 组件 | meta | 说明 |
|------|------|------|------|
| `/` | — | — | 重定向至 `/checkin` |
| `/login` | LoginView | `guestOnly: true` | 登录页 |
| `/register` | RegisterView | `guestOnly: true` | 注册页 |
| `/checkin` | CheckinView | `requiresAuth: true` | 30 天打卡页 |
| `/chat` | ChatView | `requiresAuth: true` | AI 对话（FitApp + 智能体） |
| `/chat?mode=planning` | ChatView | `requiresAuth: true` | AI 规划模式 |

### 5.2 导航守卫

```typescript
router.beforeEach(async (to) => {
  const auth = useAuthStore()

  // 首次进入：尝试恢复登录态
  if (auth.token && !auth.user) {
    try {
      await auth.fetchCurrentUser()
    } catch {
      auth.logout()
    }
  }

  if (to.meta.requiresAuth && !auth.isAuthenticated) {
    return { path: '/login', query: { redirect: to.fullPath } }
  }

  if (to.meta.guestOnly && auth.isAuthenticated) {
    return { path: '/checkin' }
  }
})
```

### 5.3 ChatView 模式分发

```
ChatView
├── route.query.mode === 'planning'
│   └── PlanningWizard          ← 独立规划流程
└── 默认
    └── ChatTabs
        ├── FitAppChat          ← Tab: 健康咨询
        └── ManusChat           ← Tab: 智能体
```

**设计决策（D-05）**：AI 规划模式采用**独立子模式**，不接入 Manus 智能体。

| 对比维度 | 独立规划模式（✅ 采用） | 接入 Manus 智能体（❌ 不采用） |
|----------|------------------------|-------------------------------|
| 需求符合度 | 固定 4 题引导，与需求 §2.3.3 完全一致 | 自由对话，无法保证固定问答顺序 |
| 后端就绪度 | `/plan/ai/*` 全套 API 已实现 | 需新增 Plan Tool + 鉴权改造 |
| 输出结构 | 后端强制校验 30 天 JSON + 4 种 type | LLM 自由输出，无法保证格式 |
| 用户隔离 | JWT + Redis 会话绑定 userId | Manus 当前无鉴权、无用户绑定 |
| 预览确认 | 专用 Preview UI，支持编辑 title | 需在聊天流中嵌入复杂表单 |
| MVP 工期 | 前端适配已有 API 即可 | 前后端均需大量改造 |

**结论**：规划模式走 `/chat?mode=planning` 路由参数，渲染 `PlanningWizard` 组件，与 FitApp/Manus Tab 完全隔离。入口仍从打卡页「AI 帮我规划」按钮跳转。

## 6. 模块详细设计

### 6.1 认证模块

#### 6.1.1 数据流

```
注册: POST /user/register → 成功 → 跳转 /login
登录: POST /user/login → 存 token + userVO → 跳转 /checkin（或 redirect 参数）
退出: POST /user/logout → 清 localStorage → 跳转 /login
恢复: GET /user/get/login → 填充 Pinia auth store
```

#### 6.1.2 Pinia Store（auth.ts）

| State | 类型 | 说明 |
|-------|------|------|
| token | `string \| null` | JWT，持久化至 localStorage |
| user | `LoginUserVO \| null` | 当前用户信息 |

| Action | 说明 |
|--------|------|
| login(account, password) | 调用 API，存 token + user |
| register(form) | 注册后跳转登录 |
| logout() | 清 token，调 API，跳转登录 |
| fetchCurrentUser() | 应用初始化 / 刷新时恢复 |

#### 6.1.3 Axios 拦截器

**请求拦截器**：从 localStorage 读取 token，注入 `Authorization: Bearer <token>`

**响应拦截器**：
- `code === 0` → 返回 `data`
- `code === 40100` → 清 token，跳转 `/login`
- 其他非 0 → `Promise.reject(message)`

> 说明：后端 JWT 有效期 7 天，MVP 不做 Refresh Token。2025–2026 生产级方案推荐 Access + Refresh 双 Token，但不在本期范围。

#### 6.1.4 表单校验（前端）

| 字段 | 规则 |
|------|------|
| userAccount | 4–16 位，字母/数字/下划线 |
| userPassword | 至少 8 位 |
| checkPassword | 与密码一致 |

### 6.3 AI 对话模块（FitApp + Manus 双模式）

#### 6.3.1 页面结构

```
┌──────────────────────────────────────────────┐
│ AppNav                                        │
├──────────────────────────────────────────────┤
│ ChatTabs:  [ 健康咨询 ]  [ 智能体 ]           │
├──────────────────────────────────────────────┤
│ ChatMessageList                               │
│   👤 用户: 跑步后膝盖疼怎么办？               │
│   🤖 FitApp: 根据你的描述，建议……             │
│   ── 或 ──                                    │
│   👤 用户: 帮我搜索最新减脂研究               │
│   🤖 Manus: step1: 正在搜索…                  │
│             step2: 工具调用 WebSearch…        │
│             step3: 已生成 PDF…                │
├──────────────────────────────────────────────┤
│ ChatInput: [输入框]  [发送/停止]              │
└──────────────────────────────────────────────┘
```

#### 6.3.2 FitApp 健康咨询模式

| 项目 | 说明 |
|------|------|
| 接口 | `GET /fitai/fitapp/chat/stream?message=&chatId=` |
| 协议 | SSE（EventSource） |
| 多轮记忆 | 前端生成 UUID 作为 chatId，存 sessionStorage |
| 特点 | RAG 运动健康知识库，专注运动健康答疑 |
| 鉴权 | 不需要 |

**Composable: useFitAppChat**

```typescript
interface FitAppMessage {
  id: string
  role: 'user' | 'assistant'
  content: string
  status: 'complete' | 'streaming'
}

// 核心流程
function send(message: string) {
  appendUserMessage(message)
  const assistantMsg = appendAssistantMessage('', 'streaming')
  const es = new EventSource(url)
  es.onmessage = (e) => { appendToMessage(assistantMsg.id, e.data) }
  es.onerror = () => { es.close(); markComplete(assistantMsg.id) }
}
```

**注意**：EventSource 仅支持 GET，与后端接口匹配。关闭连接时调用 `es.close()`。

#### 6.3.3 Manus 智能体模式

| 项目 | 说明 |
|------|------|
| 接口 | `GET /fitai/manus/chat?message=` |
| 协议 | SSE（SseEmitter，每步推送 `stepN: 结果`） |
| 工具能力 | WebSearch、WebScraping、PDF 生成、文件操作、终端、资源下载 |
| 多轮记忆 | **当前后端无 chatId 参数**，每次请求创建新 Manus 实例 |
| 鉴权 | 不需要 |

**Composable: useManusChat**

Manus SSE 每条消息格式为 `step{N}: {内容}`，前端需：
1. 解析 step 编号，展示工具调用进度
2. 用 `ToolCallBadge` 组件标识当前步骤（搜索中 / 生成 PDF 中…）
3. 检测内容中的 `PDF generated successfully to:` 字样，提取文件路径展示

**MVP 限制（见 §8.3）**：PDF 保存在服务端 `{user.dir}/tmp/pdf/`，当前无文件下载 API，前端仅展示路径文本，不提供下载按钮。

#### 6.3.4 双模式状态隔离

| 隔离项 | 策略 |
|--------|------|
| 消息历史 | FitApp 与 Manus 各自独立 messages 数组 |
| chatId | 仅 FitApp 使用，Manus 不使用 |
| Tab 切换 | 切换 Tab 不中断进行中的 SSE（允许并行，但 MVP 建议切换时 abort 当前流） |
| 输入框 | 共享 ChatInput 组件，根据 activeTab 分发 send 逻辑 |

#### 6.3.5 Pinia Store（chat.ts）

| State | 说明 |
|-------|------|
| activeTab | `'fitapp' \| 'manus'` |
| fitappMessages | FitApp 消息列表 |
| manusMessages | Manus 消息列表 |
| fitappChatId | UUID，sessionStorage 持久化 |
| isStreaming | 当前 Tab 是否正在流式输出 |

#### 6.4.2 状态机

```
                    ┌─────────────┐
                    │   ENTER     │
                    └──────┬──────┘
                           │
              ┌────────────▼────────────┐
              │  hasPlan === true ?     │

            Yes  │                  │ No
                 ▼                  ▼
         ┌──────────────┐   ┌──────────────┐
         │ ConfirmDialog│   │  QUESTION    │
         │ → replan API │   │  (Q1)        │
         └──────┬───────┘   └──────┬───────┘
                │                  │
                └────────┬─────────┘
                         ▼
              ┌─────────────────────┐
              │  QUESTION (Q1→Q4)  │◄── POST /plan/ai/chat {message}
              │  phase = question   │
              └──────────┬──────────┘
                         │ 第 4 题回答后（10~30s loading）
                         ▼
              ┌─────────────────────┐
              │  PREVIEW            │
              │  phase = preview    │◄── GET /plan/ai/preview（刷新恢复）
              └──────────┬──────────┘
                         │ 用户编辑 + 确认
                         ▼
              ┌─────────────────────┐
              │  CONFIRM            │
              │  POST /plan/ai/confirm│
              └──────────┬──────────┘
                         ▼
              ┌─────────────────────┐
              │  router.push        │
              │  ('/checkin')       │
              └─────────────────────┘
```

#### 6.4.3 固定问题（与后端 PlanConstant 一致）

| 序号 | 问题 |
|------|------|
| 1 | 你每天大概有多少时间？ |
| 2 | 你的目标是什么？（减脂 / 增肌 / 保持习惯） |
| 3 | 有没有身体限制？（选填，没有可回复「无」） |
| 4 | 需要安排饮食或作息吗？ |

#### 6.4.4 预览页交互

- 展示 30 天 × 每天 3~5 条任务
- 用户可编辑 `title`（`<input>` 内联编辑）
- `type` 不可编辑，以标签展示
- 底部「确认计划」按钮 → `POST /plan/ai/confirm`
- 请求体必须包含完整 30 天数据（dayIndex 1–30）

#### 6.4.5 会话恢复

- 规划会话存 Redis，TTL 2 小时
- 用户刷新预览页时：调用 `GET /plan/ai/preview` 恢复
- 超时后提示「规划会话已过期，请重新开始」

## 7. UI/UX 设计规范

### 7.1 设计风格

| 维度 | 方案 |
|------|------|
| 产品类型 | 运动健康 C 端工具 |
| 风格 | 简约、清爽、鼓励坚持 |
| 主色 | `#10B981`（emerald-500，健康活力） |
| 辅色 | `#6366F1`（indigo-500，AI 功能） |
| 背景 | `#F9FAFB`（gray-50） |
| 字体 | 系统字体栈：`Inter, "PingFang SC", "Microsoft YaHei", sans-serif` |
| 圆角 | `rounded-xl`（卡片）、`rounded-lg`（按钮） |
| 阴影 | `shadow-sm`（卡片）、`shadow-md`（弹窗） |

### 7.2 响应式

MVP 以 **移动端优先 + 桌面适配** 为目标：
- 主内容区 `max-w-2xl mx-auto px-4`
- 打卡页、对话页均单列布局
- 导航栏固定顶部 `sticky top-0 z-50`

### 7.3 交互状态

| 场景 | 反馈 |
|------|------|
| API 请求中 | 按钮 disabled + spinner |
| AI 生成计划（10~30s） | 全屏 loading + 文案「AI 正在生成你的 30 天计划…」 |
| SSE 流式输出 | 光标闪烁 / StreamingIndicator |
| 操作成功 | Toast 提示（2s 自动消失） |
| 操作失败 | Toast 错误提示 + 具体 message |
| 401 未登录 | 自动跳转登录页 |

### 7.4 无障碍（MVP 基线）

- 所有交互元素可键盘聚焦
- 任务 checkbox 使用原生 `<input type="checkbox">` + label
- 颜色对比度满足 WCAG AA（文本/背景 ≥ 4.5:1）

### 8.3 规划模块

| 方法 | 路径 | 前端调用场景 |
|------|------|-------------|

| 方法 | 路径 | 前端调用场景 |

### 8.5 已知限制与应对

| 限制 | 影响 | MVP 应对 |
|------|------|----------|
| Manus 无 chatId，无多轮记忆 | 智能体每轮独立 | 前端提示「智能体模式为单轮任务」 |
| PDF 存服务端 tmp 目录，无下载 API | 用户无法下载 PDF | SSE 消息中展示文件路径文本；后续版本新增 `GET /fitai/files/{name}` |
| `/fitai/**` 无需鉴权 | 未登录也能调 AI 接口 | 符合后端设计，前端路由仍要求登录后访问 `/chat` |
| 规划会话 2 小时过期 | 中断后需重来 | 过期时 Toast 提示 + 重新开始按钮 |
| confirm 请求体较大（30 天任务） | 网络传输 | 设置 Axios timeout 60s |
